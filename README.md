# 求职分析与投递 Agent

> 🤖 你的 AI 求职助手，自动搜岗、智能评估、自学习进化、越用越懂你

---

## ✨ 核心特性

### 🚀 自动化工作流
从搜索到投递全程自动化，解放你的时间：
- **一键搜岗**：按偏好自动搜索猎聘等平台
- **智能初筛**：自动去重、过滤不符合岗位
- **评分评估**：按权重模型自动打分+风险核查
- **简历定制**：岗位针对性 HTML 简历生成+PDF 导出
- **一键投递**：按你的选择批量投递

### 🔄 规则自更新
**越用越准，自动进化：**
- 你选「投」→ 自动学习正向偏好
- 你选「暂不投/pass」→ 自动提取降权因素  
- 你改规则 → 立即生效，后续全部按新规则执行
- 所有规则动态保存在 `preferences.md`，可随时查看、手动调整

### 🔍 智能探索机制
像真人猎头一样持续探索：
- **多轮搜岗**：自动切换关键词、地区维度深挖机会
- **去重过滤**：重复岗位自动跳过，不浪费你的时间
- **配额控制**：按规则控制搜索频率，不过度打扰平台
- **分层记录**：搜索结果、评分明细、投递日志自动归档

### 📝 在线可编辑简历
生成的 HTML 简历：
- ✅ 文字直接点击编辑，改完重新打印 PDF
- ✅ 照片直接 base64 嵌入，不依赖本地文件
- ✅ 蓝色条/技能条/背景色打印不丢失
- ✅ A4 精准排版，导出即完美可用

## 🎯 快速上手：3 步搞定投递

### 第 1 步：开始搜索
```text
开始搜索
```
Agent 自动按你的偏好搜岗、评分、去重、写入文件。

### 第 2 步：查看候选
```text
/投递
```
展示高匹配度岗位清单，带分数、亮点、风险。

### 第 3 步：选择并投递
```text
输入：1-5,8,10
```
选好岗位后，Agent 会：
- 询问是否生成定制简历 PDF
- 展示最终确认清单（岗位、公司、薪资、简历方式）
- **你确认后**才调用猎聘 MCP 投递
- 投递后自动记录日志+学习你的偏好

## 🔄 完整工作闭环

```
搜索岗位 → 评分评估 → 你选择 → 生成简历 → 投递 → 偏好学习
    ↑                                                        ↓
    └──────────────────────────────────────────────────────┘
         每一轮投递都让 Agent 更懂你
```

### 1. 岗位探索阶段
- 从 `preferences.md` 读取目标职位、地点、薪资、搜索配额
- 自动切换关键词多轮搜索
- 去重（已搜过、已投递、已 pass 的自动跳过）
- 按风险规则初筛（外包、培训机构、加班重灾区自动降级）

### 2. 评分评估阶段
- 按当前评分模型打分（行业、规模、薪资、地点、匹配度等维度）
- A-G 模块深度评估（公司、产品、团队、文化、成长、风险、个人匹配）
- 风险核查（试用期比例、加班文化、CEO 口碑、历史裁员等）
- 结果分层写入：摘要→job.md，详情→job_detail.md，原始搜索→job_search_archive.md

### 3. 投递执行阶段
- `/投递` 命令按规则筛选展示候选
- 可选生成岗位定制优化简历 PDF
- 投递前双重确认（岗位清单+简历方式）
- 你明确说「确认」才调用猎聘 MCP
- 投递后更新：job.md（状态）+ job_apply_log.md（日志）+ preferences.md（偏好学习）

### 4. 自更新进化阶段
**每次你做出选择，Agent 都在学习：**
- 你选「投」→ 该岗位的正向特征加入偏好库
- 你选「暂不投」→ 你说明的原因加入降权库
- 你选「pass」→ 负面特征显著，未来同类自动降权
- 你手动改 rules → 立即生效，后续按新规则

## 📁 文件架构

| 文件 | 作用 | 谁来更新 |
|---|---|---|
| `preferences.md` | **动态规则源**：偏好、评分模型、搜索规则、投递规则、学习规则 | Agent 自更新 + 你可手动改 |
| `job.md` | 岗位主索引：最新状态、评分摘要、是否投递过 | 每次搜/评/投后自动更新 |
| `job_detail.md` | 岗位详情：评分明细、A-G 模块、面试准备、风险核查 | 需要看详情时读取/更新 |
| `job_search_archive.md` | 搜索归档：原始结果、批次、去重记录 | 每次搜索后追加 |
| `job_apply_log.md` | 投递日志：时间、岗位、打招呼语、PDF 路径、结果 | 每次投递后追加 |
| `优化简历/*.html/*.pdf` | 岗位定制简历成品 | 生成优化简历时写入 |

## ⚙️ 可配置项（你可以随时改）

在 `preferences.md` 中修改，改完立即生效：

- **目标职位**：搜什么岗位、优先级
- **偏好地点**：城市、通勤、远程偏好
- **薪资偏好**：最低线、期望线、加分区间
- **加权偏好**：喜欢什么公司/行业/关键词，不喜欢什么
- **评分模型**：各维度权重、加分减分项
- **等级规则**：A/B/C/D/F 级对应的分数范围
- **搜索规则**：每轮搜多少、每天搜几轮、去重策略
- **投递规则**：/投递 展示逻辑、简历生成策略、在线简历同步

## 🛡️ 安全边界（不可突破）

无论怎么配置，以下规则永远有效：

- ❌ 未经你最终确认，绝不调用投递 MCP
- ❌ 不投递已 pass、暂不投或已投递的岗位
- ❌ 不伪造投递状态、PDF 发送状态
- ❌ 未经你确认，绝不修改猎聘在线简历
- ❌ 如需修改在线简历，必先展示修改内容让你确认

## 🔧 环境配置

### 猎聘 MCP
配置入口：`https://www.liepin.com/mcp/server#config`

配置后可用能力：
- 读取我的简历
- 搜索岗位
- 投递岗位
- 修改在线简历（修改前必确认）

### LLM API
用环境变量或 `.env` 配置：
```text
OPENAI_API_KEY=你的key
ANTHROPIC_API_KEY=你的key
MODEL_NAME=你用的模型名
```

### 简历导出 PDF 注意事项
用 Edge/Chrome 打开 HTML，按 `Ctrl+P`：
1. 展开「更多设置」
2. **务必勾选 ☑️ 打印背景图形**（否则蓝色条和技能条会消失）
3. 目标选「另存为 PDF」

## 💡 常见问题

**Q: Agent 会自动投递吗？**  
A: 不会。投递前必须你明确确认，双重确认机制。

**Q: 偏好学习是自动的吗？**  
A: 是的。你每次选投/不投/pass，Agent 自动记录并更新偏好库。

**Q: 生成的简历能改吗？**  
A: 能。HTML 打开后文字直接点击编辑，改完重新 Ctrl+P 导出 PDF。

**Q: 规则改了能重评已搜过的岗位吗？**  
A: 能。直接说「按新规则重评」即可。

**Q: 会重复搜同一个岗位吗？**  
A: 不会。自动去重，已搜过的会跳过。

## 📌 设计原则

1. **规则与代码分离**：所有业务规则在 `preferences.md`，主提示词只定义流程和安全边界
2. **数据分层存储**：按读写频率和内容类型分文件，避免上下文爆炸
3. **自反馈闭环**：用户决策 → 自动学习 → 规则更新 → 下轮更准
4. **安全优先**：涉及投递、修改在线简历等操作，必有人类确认环节
